## 脆弱性の報告 / Reporting a Vulnerability

**脆弱性は公開の Issue では報告しないでください。**
セキュリティ上の問題を発見した場合は、以下の方法でご連絡ください。

**Please do not report security vulnerabilities through public GitHub Issues.**
If you discover a security issue, please report it via one of the following methods.

### 連絡先 / Contact

| 方法 / Method | 連絡先 / Address |
| ------------- | ---------------- |
| メール / Email | [contact@daruks.com](mailto:contact@daruks.com) |
| お問い合わせフォーム / Contact Form | [https://corp.daruks.com/contact](https://corp.daruks.com/contact) |

以下の情報をできる限り含めてください：

Please include as much of the following information as possible:

- 問題の種類（例：バッファオーバーフロー、SQL インジェクション、XSS など）
- Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting)
- 問題に関連するソースファイルのパスまたは URL
- The path or URL of the source file(s) related to the issue
- 再現手順
- Step-by-step instructions to reproduce the issue
- 概念実証コードまたはエクスプロイトコード（可能な場合）
- Proof-of-concept or exploit code (if possible)
- 問題の影響範囲
- Impact of the issue

### 対応について / Response

- 報告受領後、**3日以内**に確認の返信をお送りします
- We will acknowledge receipt of your report within **3 days**
- 調査・修正の進捗について随時ご連絡します
- We will keep you informed of our investigation and remediation progress
- 問題が確認された場合、修正リリース後に報告者のクレジットを記載します（希望する場合）
- If the issue is confirmed, we will credit the reporter after the fix is released (if desired)
